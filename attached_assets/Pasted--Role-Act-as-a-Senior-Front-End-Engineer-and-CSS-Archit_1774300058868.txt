# Technical Constraints & Best Practices (CRITICAL)
1. **Design Tokens:** 
   - Define 30+ CSS custom properties (`:root`) for colors, spacing, radius, transitions, and glow values. 
   - Ensure tokens are scoped for potential future Light Mode support.
2. **Accessibility (WCAG):** 
   - **Motion:** Wrap all animations (flicker, shift, border-dance) in `@media (prefers-reduced-motion: reduce)` to disable them for sensitive users.
   - **Contrast:** Ensure gradient text meets WCAG AA contrast ratios against the background. Add subtle text-shadows if needed.
   - **Focus:** Ensure focus states are visible and distinct (min 2px width), not obscured by animations.
   - **Photosensitivity:** Ensure "flicker" frequency is safe (<3 flashes per second).
3. **Performance:** 
   - Use `will-change: transform` sparingly on animated elements. 
   - Prefer animating `transform` and `opacity` over `background-position` where possible to avoid layout thrashing.
   - Ensure fixed backgrounds do not cause scroll jank on mobile.
4. **Browser Compatibility:** 
   - Style native `<select>` dropdowns carefully; ensure options remain readable on iOS Safari (avoid white-on-white).
